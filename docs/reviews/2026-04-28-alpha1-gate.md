## Resolved Blocking Findings In Candidate Tree

- Canonical manifests enforce `protocol_id`, `root_hash`, `manifest_id`, ordering, piece coverage, and file-root `.` identity.
- Manifest paths reject escapes, dot aliases, repeated separators, colon paths, backslash aliases, symlink roots, symlinked ancestors, and special filesystem entries.
- Manifest creation, tree verification, piece reads, and piece writes use no-follow file opens for traversed components.
- Manifest validation enforces canonical `piece_size` chunking.
- Public schemas use strict typing for numeric fields.
- Tree verification performs shape checks around no-follow file inspection so post-scan symlink swaps are rejected.
- Downloader output targets reject symlink leaves and existing symlink ancestors before staging.
- Downloader parent creation revalidates every created ancestor and rejects symlink races before staging can write.
- Downloader staging uses unguessable staging names, staging identity guards, no-follow piece writes, and destination-kind checks.
- Cache promotion builds a fresh promotion root from piece-verified staging bytes, verifies it, and installs it with no-follow parent file descriptors.
- Seeder responses hash-check served piece bytes after request-time tree verification and recheck served root identity before and after the piece read.
- Tracker-backed discovery intersects advertised piece maps with manifest-declared piece IDs before using a peer.
- Empty manifests may be announced with empty piece maps, but those maps are only useful to clients validating against a zero-piece manifest.
- Tracker announces and tracker response peers accept only HTTP(S) peer URL origins and reject username/password userinfo, URL path/query/fragment components, localhost, private, link-local, unspecified, reserved, and otherwise non-global IP literals; resolvable DNS names must resolve only to globally routable addresses.
- Installed `bitswarm` console script routes through `safe_main` for controlled user-facing errors.

## Residual Explicit Alpha Constraints

- Path grammar is a conservative portable subset, not the full native filesystem grammar.
- macOS top-level compatibility symlinks such as `/tmp` and `/var` may be traversed so normal temp/cache paths work; user-controlled symlink components below that layer are rejected.
- The in-memory tracker retains peer-secret bindings for process lifetime to prevent expired-id takeover.
- If DNS resolution fails locally during tracker peer URL validation, unresolved fully-qualified hostnames may be accepted; deployments needing stricter egress guarantees must enforce resolver and IP filtering outside the alpha schema.
